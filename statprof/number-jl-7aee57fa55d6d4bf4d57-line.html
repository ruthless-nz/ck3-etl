268 """
269 one(x)
270 one(T::type)
271
272 Return a multiplicative identity for `x`: a value such that
273 `one(x)*x == x*one(x) == x`. Alternatively `one(T)` can
274 take a type `T`, in which case `one` returns a multiplicative
275 identity for any `x` of type `T`.
276
277 If possible, `one(x)` returns a value of the same type as `x`,
278 and `one(T)` returns a value of type `T`. However, this may
279 not be the case for types representing dimensionful quantities
280 (e.g. time in days), since the multiplicative
281 identity must be dimensionless. In that case, `one(x)`
282 should return an identity value of the same precision
283 (and shape, for matrices) as `x`.
284
285 If you want a quantity that is of the same type as `x`, or of type `T`,
286 even if `x` is dimensionful, use [`oneunit`](@ref) instead.
287
288 # Examples
289 ```jldoctest
290 julia> one(3.7)
291 1.0
292
293 julia> one(Int)

296 julia> import Dates; one(Dates.Day(1))
