114 reverse(s::AbstractString) -> AbstractString
115
116 Reverses a string. Technically, this function reverses the codepoints in a string and its
117 main utility is for reversed-order string processing, especially for reversed
118 regular-expression searches. See also [`reverseind`](@ref) to convert indices in `s` to
119 indices in `reverse(s)` and vice-versa, and `graphemes` from module `Unicode` to
120 operate on user-visible "characters" (graphemes) rather than codepoints.
121 See also [`Iterators.reverse`](@ref) for
122 reverse-order iteration without making a copy. Custom string types must implement the
123 `reverse` function themselves and should typically return a string with the same type
124 and encoding. If they return a string with a different encoding, they must also override
125 `reverseind` for that string type to satisfy `s[reverseind(s,i)] == reverse(s)[i]`.
126
127 # Examples
128 ```jldoctest
129 julia> reverse("JuliaLang")
130 "gnaLailuJ"
131
132 julia> reverse("ax̂e") # combining characters can lead to surprising results
133 "êxa"
134
135 julia> using Unicode
136
137 julia> join(reverse(collect(graphemes("ax̂e")))) # reverses graphemes
138 "ex̂a"
139 ```
