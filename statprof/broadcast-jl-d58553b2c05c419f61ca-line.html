271 From
272 f.(a, g.(b, c), d)

276 h(w, x, y, z) = f(w, g(x, y), z)
277 In terms of its internal representation,
278 Broadcasted(f, a, Broadcasted(g, b, c), d)
279 becomes
280 Broadcasted(h, a, b, c, d)
281
282 This is an optional operation that may make custom implementation of broadcasting easier in

285 function flatten(bc::Broadcasted{Style}) where {Style}
286 isflat(bc) && return bc
287 # concatenate the nested arguments into {a, b, c, d}
288 args = cat_nested(bc)
289 # build a function `makeargs` that takes a "flat" argument list and
290 # and creates the appropriate input arguments for `f`, e.g.,
291 # makeargs = (w, x, y, z) -> (w, g(x, y), z)
292 #
293 # `makeargs` is built recursively and looks a bit like this:
294 # makeargs(w, x, y, z) = (w, makeargs1(x, y, z)...)
295 # = (w, g(x, y), makeargs2(z)...)
296 # = (w, g(x, y), z)
297 let makeargs = make_makeargs(()->(), bc.args), f = bc.f
298 newf = @inline function(args::Vararg{Any,N}) where N
299 f(makeargs(args...)...)

513 ## Indexing manipulations
514 """
515 newindex(argument, I)
516 newindex(I, keep, default)
517
518 Recompute index `I` such that it appropriately constrains broadcasted dimensions to the source.
519
520 Two methods are supported, both allowing for `I` to be specified as either a [`CartesianIndex`](@ref) or
521 an `Int`.
522
523 * `newindex(argument, I)` dynamically constrains `I` based upon the axes of `argument`.
524 * `newindex(I, keep, default)` constrains `I` using the pre-computed tuples `keeps` and `defaults`.
525 * `keep` is a tuple of `Bool`s, where `keep[d] == true` means that dimension `d` in `I` should be preserved as is
526 * `default` is a tuple of Integers, specifying what index to use in dimension `d` when `keep[d] == false`.
527 Any remaining indices in `I` beyond the length of the `keep` tuple are truncated. The `keep` and `default`
528 tuples may be created by `newindexer(argument)`.
529 """
530 Base.@propagate_inbounds newindex(arg, I::CartesianIndex) = CartesianIndex(_newindex(axes(arg), I.I))

676 """

678

680
681 Broadcasting applies the function `f` over the elements of the container arguments and the
682 scalars themselves in `As`. Singleton and missing dimensions are expanded to match the
683 extents of the other arguments by virtually repeating the value. By default, only a limited
684 number of types are considered scalars, including `Number`s, `String`s, `Symbol`s, `Type`s,

686 iterated over or indexed into elementwise.
687
688 The resulting container type is established by the following rules:
689
690 - If all the arguments are scalars or zero-dimensional arrays, it returns an unwrapped scalar.
691 - If at least one argument is a tuple and all others are scalars or zero-dimensional arrays,
692 it returns a tuple.
693 - All other combinations of arguments default to returning an `Array`, but
694 custom container types can define their own implementation and promotion-like
695 rules to customize the result when they appear as arguments.
696

1120 # which converts all function calls and assignments into
1121 # broadcasting "dot" calls/assignments:
1122
1123 dottable(x) = false # avoid dotting spliced objects (e.g. view calls inserted by @view)
1124 # don't add dots to dot operators
1125 dottable(x::Symbol) = (!isoperator(x) || first(string(x)) != '.' || x === :..) && x !== :(:)
1126 dottable(x::Expr) = x.head != :$
1127 undot(x) = x
1128 function undot(x::Expr)
1129 if x.head == :.=
1130 Expr(:(=), x.args...)
1131 elseif x.head == :block # occurs in for x=..., y=...
1132 Expr(:block, map(undot, x.args)...)
1133 else
1134 x
1135 end
1136 end
1137 __dot__(x) = x
1138 function __dot__(x::Expr)
1139 dotargs = map(__dot__, x.args)
1140 if x.head == :call && dottable(x.args[1])
1141 Expr(:., dotargs[1], Expr(:tuple, dotargs[2:end]...))
1142 elseif x.head == :comparison
1143 Expr(:comparison, (iseven(i) && dottable(arg) && arg isa Symbol && isoperator(arg) ?
1144 Symbol('.', arg) : arg for (i, arg) in pairs(dotargs))...)
1145 elseif x.head == :$
1146 x.args[1]
1147 elseif x.head == :let # don't add dots to `let x=...` assignments
1148 Expr(:let, undot(dotargs[1]), dotargs[2])
1149 elseif x.head == :for # don't add dots to for x=... assignments
1150 Expr(:for, undot(dotargs[1]), dotargs[2])
1151 elseif (x.head == :(=) || x.head == :function || x.head == :macro) &&
1152 Meta.isexpr(x.args[1], :call) # function or macro definition
1153 Expr(x.head, x.args[1], dotargs[2])
1154 else
1155 if x.head == :&& || x.head == :||
1156 error("""
1157 Using `&&` and `||` is disallowed in `@.` expressions.
1158 Use `&` or `|` for elementwise logical operations.
1159 """)
1160 end
1161 head = string(x.head)
1162 if last(head) == '=' && first(head) != '.'
1163 Expr(Symbol('.',head), dotargs...)
1164 else
1165 Expr(x.head, dotargs...)
1166 end
1167 end
1168 end

1171

1173 (e.g. convert `f(x)` to `f.(x)`), and convert every assignment in `expr`
1174 to a "dot assignment" (e.g. convert `+=` to `.+=`).
1175
1176 If you want to *avoid* adding dots for selected function calls in
1177 `expr`, splice those function calls in with `\$`. For example,
1178 `@. sqrt(abs(\$sort(x)))` is equivalent to `sqrt.(abs.(sort(x)))`
1179 (no dot for `sort`).
1180
1181 (`@.` is equivalent to a call to `@__dot__`.)
1182

1191 3.4233600241796016
1192 ```
