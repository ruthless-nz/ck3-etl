32 """
33 AlwaysLockedST
34
35 This struct does not implement a real lock, but instead
36 pretends to be always locked on the original thread it was allocated on,
37 and simply ignores all other interactions.
38 It also does not synchronize tasks; for that use a real lock such as [`RecursiveLock`](@ref).
39 This can be used in the place of a real lock to, instead, simply and cheaply assert
40 that the operation is only occurring on a single cooperatively-scheduled thread.
41 It is thus functionally equivalent to allocating a real, recursive, task-unaware lock
42 immediately calling `lock` on it, and then never calling a matching `unlock`,
43 except that calling `lock` from another thread will throw a concurrency violation exception.
44 """
45 struct AlwaysLockedST <: AbstractLock
46 ownertid::Int16
47 AlwaysLockedST() = new(Threads.threadid())
48 end
49 assert_havelock(l::AlwaysLockedST) = assert_havelock(l, l.ownertid)
50 lock(l::AlwaysLockedST) = assert_havelock(l)
51 unlock(l::AlwaysLockedST) = assert_havelock(l)
52 trylock(l::AlwaysLockedST) = l.ownertid == Threads.threadid()
53 islocked(::AlwaysLockedST) = true

79 """
80 wait([x])
81
82 Block the current task until some event occurs, depending on the type of the argument:
83
84 * [`Channel`](@ref): Wait for a value to be appended to the channel.
85 * [`Condition`](@ref): Wait for [`notify`](@ref) on a condition.
86 * `Process`: Wait for a process or process chain to exit. The `exitcode` field of a process
87 can be used to determine success or failure.
88 * [`Task`](@ref): Wait for a `Task` to finish. If the task fails with an exception, the
89 exception is propagated (re-thrown in the task that called `wait`).
90 * [`RawFD`](@ref): Wait for changes on a file descriptor (see the `FileWatching` package).
91
92 If no argument is passed, the task blocks for an undefined period. A task can only be
93 restarted by an explicit call to [`schedule`](@ref) or [`yieldto`](@ref).
94
95 Often `wait` is called within a `while` loop to ensure a waited-for condition is met before
96 proceeding.
97 """

114 notify(condition, val=nothing; all=true, error=false)
115
116 Wake up tasks waiting for a condition, passing them `val`. If `all` is `true` (the default),
117 all waiting tasks are woken, otherwise only one is. If `error` is `true`, the passed value
118 is raised as an exception in the woken tasks.
119
120 Return the count of tasks woken up. Return 0 if no tasks are waiting on `condition`.

149 """
150 Condition()
151
152 Create an edge-triggered event source that tasks can wait for. Tasks that call [`wait`](@ref) on a
153 `Condition` are suspended and queued. Tasks are woken up when [`notify`](@ref) is later called on
154 the `Condition`. Edge triggering means that only tasks waiting at the time [`notify`](@ref) is
155 called can be woken up. For level-triggered notifications, you must keep extra state to keep
156 track of whether a notification has happened. The [`Channel`](@ref) and [`Threads.Event`](@ref) types do
157 this, and can be used for level-triggered events.
158
159 This object is NOT thread-safe. See [`Threads.Condition`](@ref) for a thread-safe version.
