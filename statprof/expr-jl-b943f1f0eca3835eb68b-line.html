111 end

113 """
114 @macroexpand
115
116 Return equivalent expression with all macros removed (expanded).
117
118 There are differences between `@macroexpand` and [`macroexpand`](@ref).
119
120 * While [`macroexpand`](@ref) takes a keyword argument `recursive`, `@macroexpand`
121 is always recursive. For a non recursive macro version, see [`@macroexpand1`](@ref).
122
123 * While [`macroexpand`](@ref) has an explicit `module` argument, `@macroexpand` always
124 expands with respect to the module in which it is called.
125 This is best seen in the following example:
126 ```julia-repl
127 julia> module M
128 macro m()
129 1
130 end
131 function f()
132 (@macroexpand(@m),
133 macroexpand(M, :(@m)),
134 macroexpand(Main, :(@m))
135 )
136 end
137 end
138 M
139
140 julia> macro m()
141 2
142 end
143 @m (macro with 1 method)
144
145 julia> M.f()
146 (1, 1, 2)
147 ```
148 With `@macroexpand` the expression expands where `@macroexpand` appears in the code (module `M` in the example).
149 With `macroexpand` the expression expands in the module given as the first argument.
150 """
151 macro macroexpand(code)
152 return :(macroexpand($__module__, $(QuoteNode(code)), recursive=true))
153 end
